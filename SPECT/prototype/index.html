CorVision for Vercel

Cardiac SPECT segmentation frontend that is actually deployable on Vercel.

The original prototype was built in Streamlit and runs local PyTorch inference. This version keeps the product experience on Vercel and moves heavy 3D inference to a separate backend when needed.

Open Playground See Architecture
Deployment reality

Why the architecture changed

The checkpoint is about 67 MB and the app depends on PyTorch. That makes direct Vercel serverless inference a poor fit. The Vercel layer should own the UI; the inference runtime should live elsewhere.

What changed

Static-first deployment

This folder now contains a Vercel-ready static frontend. It does not require Streamlit to render the product shell.

What stayed

Local research workflow

The original Python files are still here, so local Streamlit and PyTorch-based research workflows remain intact.

What is next

External inference API

When you have an inference backend, set its endpoint in config.js. The frontend will start sending uploads there.

Architecture

Recommended deployment split

01

Vercel frontend

Hosts the landing page, upload UI, result cards, and API integration logic.

02

Inference service

Runs PyTorch, loads the checkpoint, performs preprocessing, and returns JSON plus downloadable artifacts.

03

Storage layer

Optional blob/object storage for generated masks, probability maps, and audit artifacts.
